The minimal chain

Seeds → report in five commands:

python3 scripts/01_align.py --seeds my_seeds.faa --out-dir results/ python3 scripts/02_build_hmm.py --aln results/seeds_aligned.faa --out-dir results/ python3 scripts/03_search.py --hmm results/profile.hmm --db databases/inphared.faa python3 scripts/05_classify_hits.py --hits results/hits_combined.tsv --db databases/inphared.faa python3 scripts/14_report.py --proj-dir .
A script is interactive only when stdin and stdout are real terminals and you didn't pass --yes. Piped, redirected, or --yes → it runs silently with defaults and never blocks. See the Batch & HPC tab.

Batch, HPC & Methods extraction

When no human is watching — a cluster job, cron, or a reproducible re-run — make everything silent with --yes.

Whole pipeline, hands-off

python3 scripts/run_pipeline.py --seeds my_seeds.faa --db databases/inphared.faa --proj-dir my_project/ --email you@uni.edu --cpu 8 --yes

Capture commands for a Methods section

Every tool call prints a [CMD] line. Dry-run it, or grep a real log.

python3 scripts/03_search.py --hmm profile.hmm --db db.faa --explain-only
python3 scripts/run_pipeline.py ... --yes 2>&1 | tee run.log grep '[CMD]' run.log

14_report.py also writes a ready-to-paste methods.txt and a reproducibility.json audit trail.

Resume or skip steps

python3 scripts/run_pipeline.py ... --start-at 07 --skip 09 11 --yes

Shared flags

FlagEffect
--yes, -yNon-interactive: accept defaults, never prompt
--interactiveForce prompts even when stdout isn't a terminal
--no-colorDisable ANSI colour (also honours NO_COLOR)
--explain-onlyDry run: print explanations + commands, run nothing
